Cómo funciona GraphMyCode

GraphMyCode convierte cualquier repositorio en un grafo interactivo de dependencias que se ejecuta completamente en tu navegador. No hay servidor, no hay base de datos, no hay instalación. Todo el trabajo de análisis lo hace tu CPU local mediante WebAssembly.

El pipeline en 4 pasos

1

Entrada: ZIP o URL de GitHub

Sube un archivo .zip de tu proyecto o pega la URL de un repositorio público de GitHub. En el caso de GitHub, GraphMyCode descarga los archivos directamente desde raw.githubusercontent.com a tu navegador — sin intermediarios.

Se admiten hasta 250 archivos fuente. Los archivos de más de 200 KB se omiten automáticamente para garantizar un rendimiento fluido.

2

Parseo con tree-sitter (WebAssembly)

tree-sitter es un generador de parsers incremental utilizado internamente por editores como Neovim, GitHub y VS Code. GraphMyCode lo ejecuta vía WASM directamente en el navegador, sin necesidad de ningún entorno de ejecución externo.

Para cada archivo se construye un Árbol de Sintaxis Abstracta (AST). A partir del AST se extraen:

  • Declaraciones de clases, funciones, métodos e interfaces
  • Importaciones (estáticas y dinámicas)
  • Llamadas a funciones y referencias entre símbolos
  • Rutas de dependencias entre archivos

Este proceso ocurre en un Web Worker para no bloquear el hilo principal del navegador.

3

Construcción del grafo en memoria

Los símbolos extraídos se organizan en un grafo de conocimiento usando Graphology: una biblioteca de grafos orientada a JavaScript de alto rendimiento.

Cada nodo representa un símbolo (archivo, clase, función, método, interfaz). Cada arista representa una relación (importa, llama, hereda). El grafo se construye completamente en RAM — nunca se persiste ni se envía a ningún servidor.

Para la vista Semantic 3D, se calculan adicionalmente embeddings vectoriales del código usando @huggingface/transformers (también vía WASM). Los vectores se reducen a 3 dimensiones con UMAP y se agrupan por similaridad usando el algoritmo de Leiden.

4

Renderizado interactivo en el navegador

El grafo se renderiza con cuatro modos de visualización según la pregunta que quieras responder:

🕸️

Structural

Grafo 2D interactivo de archivos, clases, funciones e importaciones. Renderizado con Sigma.js + ForceAtlas2. Responde: ¿qué importa este archivo? ¿quién llama a esta función? ¿qué módulos están aislados?

🧠

Semantic 3D

Agrupa nodos por similaridad semántica del código (no por estructura de carpetas) usando embeddings WASM + UMAP + Leiden. Renderizado con Three.js / React Three Fiber. Útil para detectar lógica duplicada y medir el impacto real de un cambio.

🏙️

Technical Debt City

Visualiza el repositorio como una ciudad 3D. Cada nodo es un edificio agrupado por carpeta. La altura del edificio representa la deuda técnica (acoplamiento + complejidad ciclomática estimada). Cuanto más alto el edificio, más difícil es cambiar ese archivo.

🔥

Dependency Heatmap

Muestra el acoplamiento real entre archivos. Las dependencias bidireccionales aparecen como aristas naranjas, revelando ciclos de importación y código espagueti de un vistazo.

Lenguajes soportados

GraphMyCode soporta 12 lenguajes de programación mediante parsers tree-sitter compilados a WASM:

  • JavaScript y TypeScript (incluyendo JSX y TSX)
  • Python
  • Java
  • Go
  • Rust
  • C y C++
  • C# (.NET)
  • PHP
  • Ruby
  • Swift
